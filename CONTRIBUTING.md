# Contributing to mgkmc

First off, thank you for considering contributing to `mgkmc`!

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub issue tracker.
- Describe the bug and include steps to reproduce it.

### Suggesting Enhancements
- Open an issue to discuss your idea.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Follow the existing coding style (PEP 8).
3. Ensure your code is well-documented with docstrings (NumPy/Google style).
4. Verify your changes build and run correctly.
5. Create a pull request!

## Coding Style
- Use 4 spaces for indentation.
- Use descriptive variable names.
- Documents modules, classes, and functions.

## Documentation
We use Sphinx for documentation. If you add new features, please ensure they are documented and that the API documentation is updated.
